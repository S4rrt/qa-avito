## B-01. Сервер отклоняет price=0 как невалидное значение

**Краткое описание:** При создании объявления с `price=0` сервер возвращает 400, считая поле незаполненным, хотя ноль является корректным значением цены.

**Шаги воспроизведения:**
1. Отправить `POST /api/1/item` с телом:
```json
{
  "sellerID": 500000,
  "name": "Test",
  "price": 0,
  "statistics": { "likes": 5, "viewCount": 10, "contacts": 1 }
}
```

**Фактический результат:**
```json
HTTP 400
{"result":{"message":"поле price обязательно","messages":{}},"status":"400"}
```

**Ожидаемый результат:**
- HTTP 200. Объявление создаётся — значение `0` валидно для цены.

**Серьёзность:** Major

**Окружение:** `https://qa-internship.avito.com`, `POST /api/1/item`

## B-03. Сервер принимает объявление с отрицательной ценой

**Краткое описание:** При создании объявления с `price=-1` сервер возвращает 200 и создаёт объявление, хотя отрицательная цена семантически невалидна.

**Шаги воспроизведения:**
1. Отправить `POST /api/1/item` с телом:
```json
{
  "sellerID": 500000,
  "name": "Test",
  "price": -1,
  "statistics": { "likes": 5, "viewCount": 10, "contacts": 1 }
}
```

**Фактический результат:**
```json
HTTP 200
{"status":"Сохранили объявление - bee058a6-1159-44ed-aac0-346dcf9ceb6a"}
```

**Ожидаемый результат:**
- HTTP 400. Сервер должен отклонять отрицательные значения цены.

**Серьёзность:** Major

**Окружение:** `https://qa-internship.avito.com`, `POST /api/1/item`
