## 1) POST /api/1/item — Создать объявление

### ТК-01. Создание объявления с валидными данными

**Шаги:**
1. Отправить `POST /api/1/item` с телом:
```json
{
  "sellerID": 123456,
  "name": "Test Item",
  "price": 1000,
  "statistics": { "likes": 5, "viewCount": 100, "contacts": 3 }
}
```

**Ожидаемый результат:**
- HTTP 200.
- Тело ответа содержит поле `status` с id созданного объявления.

---

### ТК-02. Уникальность id при одинаковых данных

**Шаги:**
1. Отправить `POST /api/1/item` дважды с одинаковым телом.

**Ожидаемый результат:**
- HTTP 200 оба раза.
- Id в ответах различаются.

---

### ТК-03. Статистика сохраняется корректно

**Шаги:**
1. Отправить `POST /api/1/item` с заданными `likes`, `viewCount`, `contacts`.

**Ожидаемый результат:**
- HTTP 200.
- Значения статистики в ответе совпадают с переданными.

---

## 2) GET /api/1/item/:id — Получить объявление по идентификатору

### ТК-13. Получение существующего объявления по валидному `id`

**Предусловия:** создано объявление через `POST /api/1/item`.

**Шаги:**
1. Отправить `GET /api/1/item/{id}`.

**Ожидаемый результат:**
- HTTP 200.
- В ответе есть `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.
- Значения соответствуют созданному объявлению.

---

### ТК-14. Консистентность данных

**Предусловия:** создано объявление с заданными `name`, `price`, `sellerID`.

**Шаги:**
1. Отправить `GET /api/1/item/{id}`.

**Ожидаемый результат:**
- HTTP 200.
- `name`, `price`, `sellerId` совпадают с переданными при создании.

---

### ТК-15. Получение объявления по несуществующему `id`

**Шаги:**
1. Отправить `GET /api/1/item/00000000-0000-0000-0000-000000000000`.

**Ожидаемый результат:**
- HTTP 404.
- Возвращается ответ об отсутствии ресурса.

---

### ТК-16. Получение объявления по некорректному `id`

**Ожидаемый результат:**

---

### ТК-17. Повторное получение одного и того же объявления (идемпотентность)

**Предусловия:** создано объявление.

## E2E — Сквозные сценарии

### ТК-29. Создать → получить по id → проверить статистику

**Шаги:**
1. Отправить `POST /api/1/item`, сохранить id.
2. Отправить `GET /api/1/item/{id}`.
3. Отправить `GET /api/1/statistic/{id}`.
4. Отправить `GET /api/2/statistic/{id}`.

**Ожидаемый результат:**
- Каждый шаг возвращает HTTP 200.
- Данные на шаге 2 совпадают с переданными на шаге 1.
- Статистика на шагах 3 и 4 непустая.

---

### ТК-30. Создать несколько объявлений → все присутствуют в списке продавца

**Шаги:**
1. Отправить `POST /api/1/item` трижды с одним `sellerID`, сохранить все id.
2. Отправить `GET /api/1/{sellerID}/item`.

**Ожидаемый результат:**
- HTTP 200 на каждом шаге.
- В списке присутствуют все три созданных объявления.
